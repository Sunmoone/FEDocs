NG-CMS 简介

  • AngularJS内容管理平台

在AngularJS上构建的前端MVC分层框架，能够支持构建复杂的单页应用，局部刷新，视图数据双向绑定，动态增删改查表格。适用于B/C类客户端网站，如内容管理系统后台。（FE-Hub也是利用这个框架来编写的，可以注意到，导航简单切换都是非iFrame实现的局部刷新）

这是最初的目标，目前只实现了其中部分

  • 处理数据是这套框架的核心，如操作，校验，组织。
  • 围绕核心集成常用组件，逐步完善可用。
  • 满足UI定制化需求，例如通过配置文件。
  • 简单易用，组织清晰，利于拓展。
  • 规范化，提高开发效率。
  • 关注性能与兼容。

主要优势

1. 分离数据与视图，减轻后端工作量

后端只提供数据源，实现完全的接口化，完全从View层剔除，后端只负责输出 JSON 数据给前端，前端收到后将 JSON 对象当作 Model 根据业务需要进行各种操作。通过HTTP协议的GET 、POST、 PUT、 DELETE四种方法进行Ajax形式的数据交互。除了Web，手机 App也能通过开放的接口获取数据。

2. JS模板，前端路由，提高性能

用户进入网站时，只需在首次载入视图模板（HTML、JS、CSS），其后的所有请求都是纯JSON数据交换，不再包含HTML代码，大大减少了数据流量。传统的后端模板需要为每个页面编译完整的HTML，再发送给浏览器，不仅占用了服务器的计算资源，而且是冗余数据的重复发送。HTML视图模板将编译页面的工作交给浏览器，可以提高性能。

前端做路由Router，当点击了某个按钮需要展示新内容时，直接由前端获取并显示另一个局部HTML页面，同时调用某个RESTful API获取JSON数据填充。

3. 数据双向绑定，无刷新，局部更新

可以用来构建强大的表格应用，无刷新的将数据的增删改查实时的显示在页面。得益于AngularJS的这一特性，HTML页面是真正动态的，跟数据紧密的绑定在一起。我们也不再需要先去Remove data，再去Remove HTML node，勉强的让视图与数据吻合。

4. 自定义HTML标签，减少代码量，提高效率

像XML/Structs一样书写标签，定义一个替换模板，然后该标签会实时的编译成浏览器可识别的HTML标签。因此我们可以定义<nav> <dialog> <datepicker> 等等实用的标签组件，通过JS库扩充HTML的功能。

一个可切换的Tab标签页，只需要在HTML中这样定义，无需书写JS 或者CSS，因为这些都被写在了编译模板里。可以很简单的实现组件的复用，调用的最简化。
